O git é uma ferramenta para controle de versões do projeto e uma grande ajuda para programadores. 
Já o GitHub é uma rede social para programadores e interface grafica para facilitar a utilização do git. 

Git 	-> Software de controle de versão (VCS)
- Repositório local. (local onde o projeto está salvo no dispositivo local).	

GutHub 	-> Plataforma de rede social para programadores.
- Repositório remoto. (local onde o projeto está salvo remotamente).

----- Tags e suas funções-MarkDown
-- Clone: Clona um repositório remoto para a máquina local.
-- pull: Atualiza o repositório local com as mudanças do repositório remoto.
-- add: Adiciona o(s) arquivo(s) que foram alterados localmente para enviar para o repositório remoto.
-- commit: Prepara os arquivos adicionados ao add e adiciona um comentário para enviar junto.
-- push: envia o commit pra o repositório remoto.

	-- Formatação de texto:

		--- Título			# Título	(Quanto mais # maior o nível do título e menor ele fica, maior: # = nível 1; menor: ## = nível 2 )

		--- emojis			:nome_do_emoji:	(github.com/ikatyang -> repositório de emoji: emoji-cheat-sheet) (Não funcina no nome do issue, usar copia e cola no emojipedia.org)

 		--- Negrito			**texto** ou __texto__
		--- Itálico			*texto* ou _texto_
		--- Riscado			~~texto~~

		--- Linha			*** ou ---

		--- linha de comando 	`comando`

		--- trecho de programa	``` 
					comando
					```

		--- citação			> texto citado	(Mais facil pela opção de mais opções do texto citado)
		--- Menção			#x	(x representa o número da issue)
